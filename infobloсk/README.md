- Системно располагает однородную информацию,
- Предназначен для управления различными блоками однородной информации,
- Инфоблоки нужны для хранения контента выводимого на сайте,
- Инфоблоки управляют контентом на сайте,
- Инфоблок - это каталогизатор однородного динамического контента на сайте,
- Инфоблоки хранят данные в БД,
- Инфоблок это аналог таблицы в базе данных (но с большими возможностями),
  - например, в инфоблоке новость одна новость содержится в одной строке таблицы,
  - а столбец это свойства инфоблока (встроенные и пользовательские),
- Данные с инфоблоков на сайте выводятся с помощью компонентов,
- Один и тот же элемент инфоблока, может быть привязан к нескольким разделам,
- Информационные блоки не могут менять свой тип. И если вам нужно перенести данные из одного типа инфоблока в другой, то сделать это можно только экспортом данных и импортом их в новый инфоблок другого типа. Аналогично и с разделами и элементами,
- При создании типа инфоблока, можно отметить галочку использовать RSS, и тогда появится доступ к урлу `RSS` у инфоблоков данного типа,
- Чтобы удалить инфоблок торгового каталога, нужно его сначала превратить в обычный инфоблок.
- Простой заполненный инфоблок можно превратить в торговый каталог: https://dev.1c-bitrix.ru/learning/course/?COURSE_ID=42&LESSON_ID=12808

При настройке инфоблока у него имеется множество вкладок в том числе: Торговый каталог - параметры вкладки настраиваются только в том случае, если инфоблок необходимо настроить на работу в режиме торгового каталога. Дальнейшую работу с разделами, элементами и свойствами инфоблока рекомендуется вести в разделе `Админка > Магазин`.

Динамический контент: новости, каталог, слайдер, справочник, блог и т.п.

## Схема создания инфоблока
- продумывание структуры инфоблока,
- создание нужного типа инфоблока с настройкой параметров,
- создание самих инфоблоков с настройкой параметров,
- создание разделов - структуры внутри инфоблока,
- создание элементов,
- создание физической страницы (в случае использования комплексного компонента) или страниц (при использовании простых компонентов) и размещение на ней компонента (компонентов) с последующей настройкой параметров компонента,
- кастомизация работы компонента под потребности ТЗ сайта.

## Создание типа инфоблока
- Переходим в `Админка > Контент > Инфоблоки > Типы инфоблоков`,
- Нажать по кнопке `Добавить новый тип`,
- ID (идентификатор типа инфоблока): `hot_price` (будет также и урлом),
- Убрать галочку при условии если в инфоблоке не будут разделы,
- Укажите название типа для разных языков интерфейса в поле Название: Горячие позиции, Hot price,
- Там же вы можете переопределить слова *Разделы* и *Элементы*, например, на *Группы товаров* и *Товары*,
- На этом настройку нового типа можно закончить - нажмите кнопку `Сохранить` (пока для нового типа не создан ни один инфоблок, этот тип не будет отображаться в разделе Контент). 

При создании типа инфоблока есть вкладка дополнительно, подробнее о ней, читайте тут: https://dev.1c-bitrix.ru/user_help/content/iblock/iblock_type_edit.php

После создания инфоблока, возможно понадобится создать комплексный компонент и поставить галочку ЧПУ, чтобы открывались детальные страницы и добавился код в файл urlrewrite.php

## Создание инфоблока
После создания инфоблока, его тип нельзя будет поменять. При создании инфоблока главное указать **Название** и **Сайты** во вкладке Инфоблок. Остальные настройки можно задать по мере необходимости.

- Переходим в `Админка > Контент > Инфоблоки > Типы инфоблоков > Название типа инфоблока`,
- Жмем по кнопке: `Добавить инфоблок`,

Удаление инфоблока:
- Админка > Контент > Инфоблоки > Типы инфоблоков > Название типа инфоблока
- Нажать на бургер рядом с именем инфоблока
- Удалить

### Вкладка Инфоблок
Сначала заполняем вкладку `Инфоблок`:
  - Символьный код: `HOT_PRICE` (в рязделах и элементах, в нижнем регистре)
  - Сайты: галочка
  - Название: Горячие позиции
  - URL страницы...: https://dev.1c-bitrix.ru/learning/course/index.php?COURSE_ID=34&LESSON_ID=3579

Символьный код - последовательность букв и цифр, используемая в IT-технологиях для обращения внутри программного кода к какому-то элементу сущности программы.

URL страницы...:
- `#SITE_DIR#/` - коренвая папка сайта `/`,
- `hot_price/` - папка в которой располагается код компонента выводящий материалы инфоблока,
- `#SECTION_ID#/` - id раздела, число,
- в URL для элемента, нужно всегда включать URL раздела, если они включены,
- у разделов и элементов коды должны совпадать, например: `#SECTION_ID#/#ELEMENT_ID#` или `#SECTION_CODE#/#ELEMENT_CODE#`

Стандартно:

    #SITE_DIR#/news/                              - для инфоблока
    #SITE_DIR#/news/#SECTION_CODE#/               - для раздела
    #SITE_DIR#/news/#SECTION_CODE#/#ELEMENT_CODE# - для элемента

Каноническая ссылка позволяет избежать дублей. Даже если элемент расположить в несколько разных разделов, то можно указать основной раздел для элемента: https://dev.1c-bitrix.ru/learning/course/index.php?COURSE_ID=34&LESSON_ID=7199

### Вкладка поля и поля разделов
- Привязка к разделам: две галочки (если разделы у инфоблока разрешены)
  - Основная страница элемента среди дублей: https://dev.1c-bitrix.ru/learning/course/?COURSE_ID=34&LESSON_ID=7199

### Вкладка SEO
Шаблоны метаданных можно настроить для всего инфоблока, раздела или элемента.  
При изменении конкретного раздела или элемента, поставьте галочку `Очистить кеш вычисленных значений`.  
Для отображения заголовка из параметра `Заголовок раздела` на странице раздела, отметьте опцию `Устанавливать заголовок страницы` в компоненте, который выводит данные инфоблока.

### Вкладка Свойства
https://dev.1c-bitrix.ru/user_help/content/iblock/iblock_edit_property.php
- свойства создают в информационном блоке
- значения свойств заполняются в элементе
- показываются свойства на сайте путем настройки параметров компонента

Создание свойства:
- введите имя свойства,
- выберите тип значения свойства,
- галочка `Множ`, если значений у свойства может быть несколько
- код свойства, заглавными литинскими буквами CODE_VALUE
- при необходимости настройте `[...]`

Для торговых каталогов свойства можно создавать двумя способами.

Выводим свойства в параметрах компонента:
- в настройках компонента переходим к настройкам списка или детального просмотра
- отмечаем необходимые свойства
- выводятся только не пустые значения свойств
Для торгового каталога, вывод свойств настраивается в настройках свойства, а не компонента.

Если свойство не показывается при добавлении элемента то проверьте окно настроек форм добаления элемента: https://dev.1c-bitrix.ru/learning/course/index.php?COURSE_ID=34&LESSON_ID=1883

Остальные: по умолчанию

# Старая запись
## Создание типа инфоблока
Перед тем как создать `инфоблок`, нужно сначала создать его тип. Чтобы создать тип инфоблока, нужно перейти `Админка > Контент > Инфоблоки > Типы инфоблоков`:

- Нажать по кнопке `Добавить новый тип`
- ID (идентификатор типа инфоблока): catalog
- Russian: Каталог (название можно задавать с пробелами)
- English: Catalog
- Сохранить

Переходим в тип инфоблока `Каталог`.

- Добавить инфоблок
- Инфоблок:
    - Символьный код: `catalog` или `template_catalog_s1`
    - Сайты: галочка
    - Название: `Каталог`
- Поля:
    - Начало активности: галочка, текущие дата и время
    - Символьный код: три галочки
- Свойства:
    - Название: Цена
    - Тип: Число
    - Код: PRICE
- Поля разделов:
    - Символьный код: три галочки
- Доступ:
    - Для всех пользователей: Чтение
- Сохранить

## Разное
- Совместный просмотр разделов и элементов: `Админ > Настройки > Настройки продукта > Настройки модулей > Информационные блоки`
- номер IBLOCK_ID у инфоблока можно посмотреть в URL списка элементов
